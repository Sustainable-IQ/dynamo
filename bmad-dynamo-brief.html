DYNAMO Memory System Architecture

Memory Pipeline Architecture

Core Memory Processing Flow:

  • External LLM Query Processing: User submits question through DYNAMO interface, routed to selected external LLM (ChatGPT, Claude, etc.), with response captured and returned to internal processing pipeline
  • Question + External Response Foundation: Question and external LLM response become foundation for comprehensive memory creation across all system components
  • Six-Stage Vectorization Pipeline: User question ‚Üí External response ‚Üí Six holon responses ‚Üí Meta-holon synthesis, with each stage independently chunked, vectorized, and stored with comprehensive metadata

Internal Memory Processing Pipeline:

  • Stage 1 - Question Vectorization: User's original question chunked, vectorized, and stored with domain categorization metadata
  • Stage 2 - External Response Vectorization: External LLM response chunked, vectorized, and stored with source model metadata
  • Stage 3 - Holon Council Deliberation: Each of six holons generates specialized response based on question + external answer with individual perspective metadata
  • Stage 4 - Individual Holon Vectorization: Each holon's unique response chunked, vectorized, and stored with holon specialization metadata
  • Stage 5 - Meta-Holon Synthesis: Final integrated response created from council deliberation with consensus/tension metadata
  • Stage 6 - Final Answer Vectorization: Meta-holon synthesis chunked, vectorized, and stored with resolution status metadata

Vector Database Schema

Core Memory Entity Structure:

  • Memory Record Components: memory_id (UUID), conversation_id (UUID for grouping), memory_type (user_question/external_response/holon_response/meta_synthesis), holon_id (UUID, null for non-holon), content_text (full text), content_chunks (array for vectorization), vector_embeddings (array), timestamp (creation datetime), meta_tags (JSON categorization)
  • Relationship Mapping: Foreign key relationships between conversation threads, holon specializations, and topic evolution tracking
  • Performance Optimization: Composite indexes on meta_tags + timestamp for fast topic-based searches across large memory sets

Meta-Tag Categorization System:

  • Domain Categories: philosophy, ethics, history, media, conspiracy, art, literature, science, ai, nutrition, health, relationships, work, personal_growth, creative, spiritual, family, education, decision_making
  • Resolution Status: resolved, unresolved, dismissed, in_progress, needs_followup, archived
  • Processing Metadata: complexity_level (simple/moderate/complex/profound), emotional_tone (neutral/positive/negative/conflicted/curious), confidence_level (high/medium/low/uncertain), memory_importance (critical/important/moderate/minor)
  • Interaction Tracking: user_interaction (accepted/questioned/rejected/saved_for_later), holon_specialization tags, consensus_level (unanimous/majority/split/unresolved)
