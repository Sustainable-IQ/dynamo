1. Multi-Agent Council Architecture

Five Specialized AI Agents

  • Authority Agent: Institutional consensus and established knowledge
  • Iconoclast Agent: Challenges assumptions and surfaces contradictions
  • Strategist Agent: Long-term consequences and systemic implications
  • Archivist Agent: Retrieves relevant context from conversation history
  • Secretariat Agent: Synthesizes all perspectives into actionable response

Dual-Ledger System

Full Ledger: Complete agent deliberations preserved for transparency and debugging

User Ledger: Clean synthesis for actionable use

Users can toggle between synthesized view and full deliberation at any time.

Critical Design Principle: A + B, Not A vs B

The multi-agent council is an additive feature, not a replacement for standard LLM chat. Users can always choose to:

  • Use the Council - Get multi-perspective analysis with full deliberation transparency
  • Bypass to External LLM - Direct access to Claude, ChatGPT, or other providers
  • Toggle Between Both - Council for complex decisions, external LLM for quick queries

Value proposition: "Everything you already have with ChatGPT/Claude, PLUS multi-agent deliberation when you want it."

2. Living Memory System

Persistent Conversation Storage: Every utterance vectorized and stored with rich metadata. Per-user Qdrant collections on your infrastructure.

Semantic Search: Natural language queries across all conversations. Hybrid retrieval (dense semantic + sparse keyword). Average search time: <3 seconds.

Resolution Tracking: Mark conversations as open/closed. Track personal evolution and growth over time.

3. Human+AI Collaboration Platform

Three Integrated Dashboards

Main Dashboard: Three-panel council chat interface with memory thread browser, council deliberation, and community activity feed.

Memory Retrieval Dashboard: Semantic search across all conversations with multi-dimensional filtering and export functionality.

Community Dashboard: Multi-user project coordination with Living Threads (Dialogue, Synthesis, Decisions, Questions views).

MVP Security Architecture

Encryption

  • At Rest: Full-disk encryption (LUKS) for PostgreSQL and Qdrant storage volumes
  • In Transit: TLS 1.3 for all client-server and server-LLM communications
  • Performance Impact: <100ms overhead, imperceptible to users

Access Controls

  • Role-based access for platform operators (read-only, admin, developer)
  • Principle of least privilege for team members
  • Multi-factor authentication for all admin accounts

Audit Logging

  • Immutable audit trail of all database access
  • Tracks: who accessed what data, when, from where, why
  • 90-day retention for MVP, 7-year retention for enterprise

Data Governance

  • Users can export complete conversation history (JSON, Markdown)
  • Users can delete all data (hard delete, not soft)
  • Clear Terms of Service regarding platform operator access rights
  • No data used for model training without explicit opt-in

Phase 2: Two-Track Security Strategy
